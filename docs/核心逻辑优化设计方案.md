### 2.3 状态转换条件

| 当前状态 | 目标状态 | 触发条件 | 说明 |
|----------|----------|----------|------|
| NORMAL | SUSPICIOUS | 1秒内请求超过阈值（如30次/秒） | 进入可疑状态，开始重点关注 |
| SUSPICIOUS | ATTACKING | 持续异常超过N秒，或请求多样性增加 | 确认为攻击 |
| SUSPICIOUS | NORMAL | 异常行为停止超过M秒 | 误判，恢复正常 |
| ATTACKING | DEFENDED | 执行防御措施（黑名单封禁） | 防御生效 |
| DEFENDED | COOLDOWN | 攻击停止（无新请求）超过X秒 | 进入冷却观察 |
| COOLDOWN | NORMAL | 冷却期结束（动态时长） | 完全恢复正常 |
| COOLDOWN | ATTACKING | 冷却期内再次攻击 | 重新进入攻击状态 |
| **ANY** | **NORMAL** | **人工操作 / 系统重置** | **强制重置状态** |

### 2.6 冷却期动态时长（新增）

#### 2.6.1 动态时长计算公式

```
cooldownDuration = baseDuration + (attackIntensity * intensityMultiplier)

其中：
- baseDuration: 基础冷却时长（默认3分钟）
- attackIntensity: 攻击强度（0-100，基于置信度、持续时间、请求量等）
- intensityMultiplier: 强度系数（默认0.5，即每增加1点强度增加0.5秒）

计算示例：
- 轻度攻击（强度=20）：3分钟 + 20*0.5秒 = 3分10秒
- 中度攻击（强度=50）：3分钟 + 50*0.5秒 = 3分25秒
- 重度攻击（强度=80）：3分钟 + 80*0.5秒 = 3分40秒
- 极端攻击（强度=100）：3分钟 + 100*0.5秒 = 3分50秒（不超过最大值10分钟）
```

#### 2.6.2 攻击强度计算

```java
/**
 * 攻击强度计算器
 */
public class AttackIntensityCalculator {
    
    /**
     * 计算攻击强度（0-100）
     */
    public int calculate(AttackContext context) {
        int intensity = 0;
        
        // 1. 置信度贡献（最高40分）
        intensity += context.getConfidence() * 0.4;
        
        // 2. 持续时间贡献（最高30分）
        long durationSeconds = context.getDuration() / 1000;
        intensity += Math.min(30, durationSeconds / 10 * 3);
        
        // 3. 请求量贡献（最高20分）
        int requestCount = context.getRequestCount();
        intensity += Math.min(20, requestCount / 100 * 5);
        
        // 4. URI多样性贡献（最高10分）
        int uriDiversity = context.getUniqueUriCount();
        intensity += Math.min(10, uriDiversity * 2);
        
        return Math.min(100, intensity);
    }
}
```

#### 2.6.3 动态冷却时长实现

```java
/**
 * 冷却时长计算器
 */
public class CooldownDurationCalculator {
    
    private long baseDuration = 180_000;      // 3分钟
    private long maxDuration = 600_000;       // 10分钟
    private double intensityMultiplier = 0.5; // 每点强度增加0.5秒
    
    /**
     * 计算冷却时长
     */
    public long calculate(AttackContext context) {
        int intensity = new AttackIntensityCalculator().calculate(context);
        
        long dynamicDuration = baseDuration + (long)(intensity * intensityMultiplier * 1000);
        
        // 不超过最大值
        return Math.min(maxDuration, dynamicDuration);
    }
    
    /**
     * 重复攻击加成
     * 如果IP有历史攻击记录，冷却时长翻倍
     */
    public long calculateWithHistory(AttackContext context, int attackHistoryCount) {
        long baseResult = calculate(context);
        
        // 每次历史攻击增加20%，最多翻倍
        double multiplier = 1.0 + Math.min(1.0, attackHistoryCount * 0.2);
        
        return Math.min(maxDuration, (long)(baseResult * multiplier));
    }
}
```

## 三、流量采集与推送重构方案

### 3.1 核心设计原则

1. **数据真实性**：保留真实请求信息，不伪造数据
2. **时序正确性**：使用队列机制保证推送顺序
3. **聚合合理性**：按 IP + 状态 + URI模式 分组聚合
4. **样本保留**：保留关键样本用于分析

        retryQueue.offer(retryTask);
    }
    
    /**
     * 异步消费线程
     */
    public void startConsumer() {
        // 单独线程消费重试队列，不阻塞网关主流程
    }

 * 重试任务

public class RetryTask implements Comparable<RetryTask> {

    private long nextRetryTime;
    private Exception lastError;

## 四、置信度计算方案

### 4.1 置信度计算公式

```
confidence = baseScore 
           + frequencyScore    // 频率异常程度
           + diversityScore    // 攻击多样性
           + persistenceScore  // 持续时间
           + patternScore      // 攻击模式匹配
           - normalBehaviorScore; // 正常行为抵扣

// 结果范围：0-100
```

### 4.2 各项评分细则

| 评分项 | 满分 | 计算规则 |
|--------|------|----------|

### 4.3 置信度平滑机制（新增）

#### 4.3.1 问题场景

```
场景：瞬间RPS暴涨又暴跌
- 0-1秒：RPS=100（攻击）
- 1-2秒：RPS=5（正常）
- 2-3秒：RPS=100（攻击）
- 3-4秒：RPS=5（正常）

问题：置信度会剧烈跳变，影响判断稳定性
```

#### 4.3.2 平滑策略

**策略一：只升不降（推荐）**

```
规则：
- 置信度只能上升，不能下降
- 直到进入 COOLDOWN 或人工重置时才清零

优点：
- 避免攻击者通过间歇性攻击绕过检测
- 判定更加稳定

缺点：
- 可能导致误判后置信度无法自动恢复
```

**策略二：滑动平均**

```
规则：
- 使用指数加权移动平均（EWMA）
- newConfidence = α * currentScore + (1-α) * previousConfidence
- α 值建议 0.3-0.5

优点：
- 平滑波动，避免毛刺
- 仍能反映趋势变化

缺点：
- 响应速度稍慢
```

#### 4.3.3 平滑机制实现

```java
/**
 * 置信度平滑处理器
 */
public class ConfidenceSmoother {
    
    // 平滑策略：ONLY_UP / SLIDING_AVERAGE
    private String smoothStrategy = "ONLY_UP";
    
    // 滑动平均系数（仅 SLIDING_AVERAGE 策略使用）
    private double alpha = 0.4;
    
    // 上一次置信度
    private Map<String, Integer> lastConfidence = new ConcurrentHashMap<>();
    
    /**
     * 平滑处理
     */
    public int smooth(String ip, int rawConfidence) {
        Integer last = lastConfidence.get(ip);
        if (last == null) {
            lastConfidence.put(ip, rawConfidence);
            return rawConfidence;
        }
        
        switch (smoothStrategy) {
            case "ONLY_UP":
                // 只升不降
                int result = Math.max(rawConfidence, last);
                lastConfidence.put(ip, result);
                return result;
                
            case "SLIDING_AVERAGE":
                // 滑动平均
                int smoothed = (int) (alpha * rawConfidence + (1 - alpha) * last);
                lastConfidence.put(ip, smoothed);
                return smoothed;
                
            default:
                return rawConfidence;
        }
    }
    
    /**
     * 重置置信度（COOLDOWN结束时调用）
     */
    public void reset(String ip) {
        lastConfidence.remove(ip);
    }
}
```
